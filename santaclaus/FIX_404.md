# Исправление ошибки 404 на Vercel

## Проблема
После деплоя на Vercel появляется ошибка 404: NOT_FOUND

## Решение

### 1. Проверьте структуру проекта
Убедитесь, что в корне папки `santaclaus` есть:
- ✅ `index.html`
- ✅ `vercel.json`
- ✅ `api/` (папка с serverless functions)
- ✅ `styles/` (папка с CSS)
- ✅ `scripts/` (папка с JS)

### 2. Обновите конфигурацию Vercel

Файл `vercel.json` уже обновлен. Он содержит:
- Rewrite правило для SPA (Single Page Application)
- Исключение для API endpoints и статических файлов

### 3. Передеплойте проект

#### Через Vercel CLI:
```bash
cd santaclaus
vercel --prod
```

#### Через Vercel Dashboard:
1. Зайдите на https://vercel.com
2. Откройте ваш проект
3. Перейдите в "Settings" → "General"
4. Убедитесь, что:
   - **Root Directory**: `santaclaus` (или оставьте пустым, если деплоите всю папку)
   - **Framework Preset**: Other
   - **Build Command**: (оставьте пустым)
   - **Output Directory**: `.` (точка)
5. Нажмите "Redeploy" в разделе "Deployments"

### 4. Проверьте настройки проекта

В Vercel Dashboard → Settings → General:

- **Framework Preset**: `Other` или `Other (no framework)`
- **Root Directory**: Если проект в подпапке, укажите `santaclaus`
- **Build Command**: (оставьте пустым)
- **Output Directory**: `.` (точка)
- **Install Command**: (оставьте пустым, если нет package.json)

### 5. Альтернативное решение

Если проблема сохраняется, попробуйте создать `package.json`:

```json
{
  "name": "santaclaus",
  "version": "1.0.0",
  "scripts": {
    "build": "echo 'No build step required'"
  }
}
```

### 6. Проверьте логи деплоя

В Vercel Dashboard:
1. Откройте ваш проект
2. Перейдите в "Deployments"
3. Откройте последний деплой
4. Проверьте "Build Logs" на наличие ошибок

### 7. Если ничего не помогает

Попробуйте упрощенную конфигурацию `vercel.json`:

```json
{
  "rewrites": [
    {
      "source": "/(.*)",
      "destination": "/index.html"
    }
  ]
}
```

## Частые причины 404:

1. ❌ Неправильный Root Directory
2. ❌ Отсутствует index.html в корне
3. ❌ Неправильная конфигурация vercel.json
4. ❌ Проект деплоится из неправильной папки

## После исправления:

После успешного деплоя ваш сайт должен быть доступен по адресу:
`https://ваш-проект.vercel.app`
